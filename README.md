# HighSim Annotated Highway Trajectory Dataset

A frame-level highway vehicle trajectory dataset derived from the HighSIM dataset and enriched with unsupervised anomaly annotations generated by the proposed **Utility-enhanced GRU-AE-GMM** framework.

---

## Overview

| Property | Value |
|---|---|
| Total records | 7,568,647 |
| Total vehicles | 2,182 |
| Unique frames | 68,701 |
| Frame ID range | 138,000 – 206,700 |
| Lane numbers | 0, 1, 2, 3 |
| Sampling rate | 30 Hz |

Each row represents **one vehicle at one frame**. Multiple vehicles are present per frame.

## Dataset Characteristics

The dataset preserves detailed vehicle interaction information, including speed, acceleration, lane position, and car-following relationships. These variables support both microscopic traffic behavior analysis and trajectory-based anomaly detection research.

The `Anomaly` column provides frame-level anomaly annotations generated by the proposed Utility-enhanced GRU-AE-GMM framework.

| Label | Meaning |
|---|---|
| 0 | Normal driving behavior |
| 1 | Anomalous driving behavior |

| Label | Count | Ratio |
|---|---|---|
| 0 (normal) | 7,554,072 | 99.81% |
| 1 (anomaly) | 14,575 | 0.19% |

The anomaly labels are generated automatically using an unsupervised learning framework rather than manual human labeling.

Unlike traditional threshold-based methods that mainly focus on extreme acceleration or braking, the proposed framework captures multi-dimensional abnormal driving patterns by jointly considering:

- speed deviation
- longitudinal-control behavior
- lane-changing behavior
- interaction risk with surrounding vehicles

As a result, the detected anomalies may include subtle or complex abnormal driving behaviors that are difficult to identify using fixed kinematic thresholds alone.

---

## Anomaly Detection Pipeline

The anomaly detection framework consists of four main stages:

1. Trajectory preprocessing and feature extraction
2. Utility-based behavioral feature construction
3. GRU-based Autoencoder training for normal behavior learning
4. Gaussian Mixture Model (GMM) clustering and adaptive threshold estimation

The framework first transforms raw trajectory variables into utility-based behavioral features representing:

- speed deviation
- longitudinal-control deviation
- lane deviation
- interaction risk

A GRU-based Autoencoder is then trained using normal driving sequences. The learned latent representations are further modeled using a Gaussian Mixture Model (GMM) to capture heterogeneous normal driving modes.

For each sequence, the Squared Prediction Error (SPE) is compared with the corresponding cluster-specific Upper Control Limit (UCL). Frames whose SPE exceeds the adaptive threshold are labeled as anomalous.
